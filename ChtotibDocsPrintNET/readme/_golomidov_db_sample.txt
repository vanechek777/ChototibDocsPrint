Системный анализ структуры базы данных
В проекте используется реляционная база данных, то есть она выполнена в виде таблиц, которые отражают сущности и полей, которые отображают атрибуты этих сущностей. Все таблицы имеют первичный ключ, по которому они и соединяются внутри базы. Также база имеет каскадное обновление (удаление), чтобы избежать риск «висячих» записей в базе.
Основной таблицей в базе данных, откуда берется начало является таблица «User» (Пользователь). Она содержит информацию о данных пользователя (полное имя, почта, телефон), его месте в организации (отдел, должность), статусе его учетной записи, хэше его пароля, а также здесь есть флаг, который прямо обозначает, является ли пользователь администратором. Все поля данной сущности представлены в таблице 1.

Вся информация о документах представлена сущностью «Document», представленной в таблице 2. Здесь указываются два внешних ключа – первый на таблицу с шаблонами. Он определяет, какой тип документа используется, а второй – на таблицу с пользователями. Он отображает идентификационный номер пользователя, который создал данный документ. Также здесь указывается статус документа, его хэш, для быстрого получения информации о нем при проверке электронной подписи. Далее идет относительный путь к документу, который указывает его месторасположение на сервере. Данная реализация позволяет быстро обращаться к документам при скачивании их с сервера. И заключительное поле – дата создания.
