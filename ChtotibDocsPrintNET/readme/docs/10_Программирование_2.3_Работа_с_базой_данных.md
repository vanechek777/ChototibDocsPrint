# 2. Программирование (продолжение)

## 2.3. Работа с базой данных

Все учётные сведения информационной системы «ЧТОТиБ Печать» хранятся в реляционной базе ChtotibDocPrint на Microsoft SQL Server (структура таблиц и связей – в п. 1.3 и 1.6). Прикладная программа не дублирует списки студентов на диске пользователя: при открытии «Группы», «Студенты», «Печать документов» и «Главная» данные читаются с сервера, а правки из карточек и диалогов снова записываются в те же таблицы. Реализация запросов и алгоритмов доступа раскрыта в п. 2.2 (листинги 2.1–2.5); ниже – сценарии развёртывания и повседневной работы с данными.

Первичное создание базы выполняет администратор скриптом CreateDatabase.sql в SQL Server Management Studio. В «Настройках» указывают строку подключения, проверяют её и сохраняют. Справочники (специальности, предметы, ГЭК) ведутся в «Админ-панели»; группы и студенты – в соответствующих разделах; оценки и реквизиты диплома – в карточке выпускника. Импорт из Excel, CSV и мастер 1С ускоряют наполнение базы перед выпускной кампанией; «Очистить сиротские записи» удаляет оценки и дипломы без привязки к существующему студенту.
