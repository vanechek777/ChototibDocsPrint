## 1.2. Описание задачи

В данном подразделе приводится техническое задание на разработку информационной системы «ЧТОТиБ Печать»: общие сведения о системе и предметной области, назначение, функциональные и нефункциональные требования. Формулировки согласованы с введением и постановкой задачи (п. 1.1).

### 1.2.1. Общие сведения

Информационная система «ЧТОТиБ Печать» (далее – ИС) – настольное прикладное программное обеспечение для автоматизации подготовки к печати документов об образовании: диплома о среднем профессиональном образовании и приложения к нему. ИС предназначена для использования в государственном профессиональном образовательном учреждении (техникуме) сотрудниками учебной части, ответственными за оформление и выдачу документов выпускникам.

Диплом считается основным документом, подтверждающим получение образования; приложение – документом, содержащим сведения об освоенных дисциплинах, часах, видах учебной деятельности и итоговых оценках. Подложка бланка – графический шаблон (изображение), на который программно накладываются текстовые поля и QR-код. Раскладка печати – совокупность координат, размеров шрифта и параметров кропа предпросмотра для лицевой и оборотной сторон диплома и приложения; настройки сохраняются в конфигурационном файле и используются при каждой печати.

ИС работает на персональном компьютере под управлением ОС Microsoft Windows. Данные об учебных группах, студентах, специальностях, предметах, оценках, реквизитах диплома и членах государственной экзаменационной комиссии (ГЭК) хранятся в реляционной базе данных ChtotibDocPrint на платформе Microsoft SQL Server. Пользователь взаимодействует с системой через графический интерфейс (главная страница, группы, студенты, печать документов, администрирование, настройки).

### 1.2.2. Назначение

Назначением ИС является сокращение трудозатрат и снижение числа ошибок при массовой подготовке дипломов и приложений в период выпускной кампании. Система должна обеспечивать централизованное хранение учебных и печатных данных, подстановку сведений о выпускнике в утверждённые макеты, предварительный просмотр результата, формирование файлов PDF и отправку комплекта на принтер.

ИС не заменяет учётную систему техникума (в том числе 1С), а дополняет её: допускается импорт справочников и списков из файлов Excel, CSV или через мастер подключения к 1С, после чего ответственный сотрудник уточняет данные и выполняет печать в едином окне приложения.

### 1.2.3. Функциональные требования

К системе предъявляются следующие функциональные требования.

1. Ведение справочника специальностей: просмотр, добавление, редактирование и удаление записей; импорт из файла.

2. Ведение учебных групп: привязка к специальности, год зачисления, курс, признак выпускающей группы; просмотр состава группы.

3. Ведение сведений о студентах: ФИО, дата рождения, сведения о предыдущем образовании, регистрационный номер, признаки выпуска и отчисления; добавление и редактирование карточки студента.

4. Учёт оценок по дисциплинам (таблицы Grades, Subjects) и реквизитов диплома (серия, номер, тип документа – обычный, с отличием, дубликат, дата выдачи) для каждого выпускника.

5. Ведение справочника предметов и членов ГЭК в разделе администрирования; фильтрация предметов по специальности; импорт специальностей и предметов из файлов.

6. Настройка подключения к СУБД: строка подключения, проверка доступности, сохранение параметров; выполнение инструкции по первичному развёртыванию базы (скрипт CreateDatabase.sql).

7. Импорт данных из внешних источников: специальности, предметы, группы, студенты – из Excel (.xlsx) или CSV; мастер подключения к 1С (OData или выгрузка в файл) с пошаговой подсказкой.

8. Раздел печати документов: выбор выпускающей группы и одного или нескольких студентов; переключение вида документа (диплом – лицевая/оборотная сторона, приложение – лицевая/оборотная сторона); предпросмотр с подложкой.

9. Настройка раскладки печати: изменение положения и ширины текстовых блоков (в том числе надписи «Дубликат» отдельно для лицевой и оборотной стороны диплома); направляющие для центрирования; настройка кропа области предпросмотра; отдельные параметры для лицевой стороны приложения (масштаб шрифта, направляющие); сохранение и загрузка настроек.

10. Формирование печатного результата: наложение текста на подложку; формирование QR-кода на обороте диплома (ФИО, код демо-экзамена, серия и номер диплома); выбор шаблонов подложек для выпускника с отличием и для дубликата; режим печати только текста без подложки.

11. Вывод документов: сохранение в PDF (отдельные стороны, приложение, полный комплект на одного студента); печать на установленный принтер; пакетная обработка выбранных студентов.

12. Обслуживание базы данных: удаление «сиротских» записей (оценки, дипломы, история печати без связи с существующим студентом или предметом).

13. Настройки интерфейса и организации: тёмная тема, размер шрифта в таблицах, наименование организации и директора для вывода в документах.

14. Учёт факта печати (таблица PrintHistory): тип документа, дата и статус операции для контроля повторной печати.

### 1.2.4. Нефункциональные требования

К системе предъявляются следующие нефункциональные требования.

1. Платформа: клиентское приложение на C# (.NET 9), интерфейс WPF; ОС – Windows 10 и выше (64-разрядная).

2. СУБД: Microsoft SQL Server или SQL Server Express / LocalDB; целостность данных обеспечивается внешними ключами между таблицами Specialties, Groups, Students, Subjects, Grades, Diplomas, CommissionMembers, PrintHistory, PrintSettings.

3. Удобство использования: навигация по основным разделам с боковой панели и с главной страницы; сообщения об ошибках подключения и импорта в понятной форме; минимальная подготовка пользователя (инструкция по БД в настройках).

4. Производительность: формирование предпросмотра и PDF для одного студента – без заметной задержки на рабочей станции техникума; пакетная печать группы допускает последовательную обработку без потери данных при сбое на отдельном студенте (с уведомлением пользователя).

5. Надёжность: при недоступности БД приложение не должно завершаться аварийно; критические операции (удаление справочников, очистка БД) подтверждаются диалогом.

6. Сопровождаемость: разделение на слои (модели, доступ к данным, сервисы печати и PDF, представление по шаблону MVVM); настройки раскладки вынесены в отдельный конфигурационный файл (diploma_print_layout.json).

7. Безопасность: работа в локальной сети техникума; учётные данные SQL Server – по политике организации (встроенная аутентификация Windows или учётная запись SQL); разграничение ролей пользователей в первой версии не реализуется (доступ к ПК ответственного сотрудника).
