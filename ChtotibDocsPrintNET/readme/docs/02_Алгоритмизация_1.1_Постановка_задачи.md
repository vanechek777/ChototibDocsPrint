# 1. Алгоритмизация

Перед началом разработки проекта требуется определить цель и границы системы, зафиксировать требования к данным и функциям, выбрать средства разработки и спроектировать структуру базы данных и приложения. Данный раздел описывает постановку задачи, требования к системе «ЧТОТиБ Печать», организацию хранения данных, программный стек, библиотеки, схему БД и архитектуру решения. Он является основой для раздела «Программирование», в котором раскрывается реализация отдельных модулей.

## 1.1. Постановка задачи

Целью моего проекта является создание прикладного программного обеспечения «ЧТОТиБ Печать», которое автоматизирует подготовку к печати дипломов и приложений к ним для выпускников техникума: подстановку данных выпускника, наложение текста на утверждённые бланки, предпросмотр, формирование PDF и вывод на принтер.

Основная задача – разработать настольное приложение под ОС Windows, отвечающее потребностям учебной части. Она включает в себя следующие этапы.

Сбор исходных данных. Необходимо определить, какие сведения хранятся в базе (группы, студенты, специальности, оценки, даты выдачи, председатель ГЭК и др.), и как они попадают в систему (ручной ввод, импорт из 1С).

Проектирование базы данных. На основе предметной области строится реляционная модель в Microsoft SQL Server с возможностью резервного копирования и обслуживания.

Проектирование интерфейса и раскладки печати. Разрабатываются экраны учёта и печати, задаются координаты надписей на лицевой и оборотной сторонах диплома и приложения, настройки кропа предпросмотра и шаблоны подложек (в том числе для дубликата и отличников).

Реализация логики формирования документов. Программная генерация PDF (диплом, приложение, полный комплект), QR-код на обороте диплома, опциональная печать без подложки.

Тестирование и подготовка к эксплуатации. Проверка сценариев печати для группы и отдельных студентов, оценка готовности к использованию в выпускную кампанию.

Ожидаемым результатом внедрения является сокращение времени на подготовку одного комплекта документов, уменьшение числа ошибок в реквизитах и расположении полей, а также возможность повторно использовать сохранённые настройки раскладки от года к году.
